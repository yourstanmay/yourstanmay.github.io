    ============üü¢ BASIC QUERIES üü¢=========================
    1. Fetch All Records from a Type
    
        SELECT * FROM {Product}
    
    2. Fetch Specific Attributes
     
        SELECT {code}, {name} FROM {Product}
    
    3. Apply a WHERE Clause
     
        SELECT {code} FROM {Product} WHERE {code} = '300460184'
    
    4. Operator : AND, OR, LIKE 
       
       SELECT {code} FROM {Product} WHERE {code} = '300460184' AND {catalogversion} = 8796093153881
       
       SELECT {code} FROM {Product} WHERE {code} = '300460184' OR {catalogversion} = 8796093153881
       
       SELECT {code} FROM {Product} WHERE {code} LIKE '30046%'
       SELECT {code} FROM {Product} WHERE {code} LIKE '%60184'
       SELECT {code} FROM {Product} WHERE {code} LIKE '%04601%'
    
    5. Using Aliases (Improves readability) and DISTINCT
     
        SELECT {p.code}, {p.name} FROM {Product AS p} WHERE {p.code} = '300460184'
        SELECT {p.code} CODE, {p.name} "PRODUCT NAME" FROM {Product AS p} WHERE {p.code} = '300460184'
        
        SELECT DISTINCT {p.code}, {p.name} FROM {Product AS p} WHERE {p.code} = '300460184'
    
    ============üü° INTERMEDIATE QUERIES üü°=========================
    1. JOIN
     
        SELECT {o.code}, {u.uid}
        FROM {Order AS o JOIN User AS u ON {o.user} = {u.pk}}	
    
    2. Use of Subquery (Use for nested queries for filtered results)
     
        SELECT {code}
        FROM {Product}
        WHERE {approvalStatus} IN ({{
            SELECT {pk} FROM {ArticleApprovalStatus} WHERE {code} = 'approved'
        }})
    
    3. ORDER BY
     
        SELECT {code} FROM {Product} ORDER BY {code} ASC
     
        SELECT {code} FROM {Product} ORDER BY {code} DESC
        
    4. LIMIT in FlexibleSearch
        üìç In HAC (Hybris Admin Console):
        LIMIT and OFFSET are not supported in the query.
    
        Use the Max. count and Start from boxes below the query input to control result size and pagination.
    
        üìç In Java Code:
    
        FlexibleSearchQuery query = new FlexibleSearchQuery(QUERY);
        query.setCount(10); // Acts like LIMIT
        query.setStart(20); // Acts like OFFSET (optional)
        
        Use ORDER BY for consistent results when paginating.
    
    ============üî¥ ADVANCED QUERIES üî¥=========================
    1. Group By and Aggregate Functions
     
        SELECT {p.catalogVersion}, COUNT({p.pk}) AS productCount
        FROM {Product AS p}
        GROUP BY {p.catalogVersion}
    
    2. Using Relations (Many-to-Many / One-to-Many)
     
        SELECT {p.code}, {c.code}
            FROM {Product AS p 
                  JOIN CategoryProductRelation AS rel ON {p.pk} = {rel.target}
                  JOIN Category AS c ON {rel.source} = {c.pk}}
            WHERE {p.code} = '300460184'
            
    
    3. FlexibleSearch with Enumeration
     
        SELECT {p.code}
        FROM {
            Product AS p 
            JOIN EnumerationValue AS e ON {p.approvalStatus} = {e.pk}
        }
        WHERE {e.code} = 'approved'
    
    
    4. Querying Localized Attributes
     
        SELECT {name[en]} "EnglishName",{name[ja]} "JapaneseName"  FROM {Product}
